GitHub Pages Hosting Guide
Publish your portfolio free at username.github.io using terminal commands (easiest!) or web browser

⚡ Terminal Method (Copy-Paste 6 Commands)

1
Create GitHub Account
Go to github.com and sign up for free. Use your real name as username (e.g., johnsmith). This becomes your portfolio URL: johnsmith.github.io
2
Install Git on Your Computer
Git is a tool to upload files to GitHub.
For Windows:
1. Download: git-scm.com/download/win
2. Run installer, click Next until done
3. Restart your computer
For Mac:
brew install git
For Linux:
sudo apt install git
3
Create Personal Access Token (Authentication)
This is like a password to upload files from terminal.
Step by step:
  1. Go to github.com/settings/tokens
  2. Click "Generate new token" → "Generate new token (classic)"
  3. Name it "Portfolio Upload"
  4. Check ONLY this box: repo
  5. Click "Generate token"
  6. Copy the long code shown - save it somewhere safe (you won't see it again!)
4
Create Repository on GitHub
A repository is a folder to store your portfolio files.
Using Web Browser:
  1. Click + → "New repository"
  2. Name: YOUR-USERNAME.github.io (replace YOUR-USERNAME)
  3. Set to Public
  4. ❌ Do NOT check "Add a README file" (this prevents conflicts)
  5. Click Create repository
5
Download Your Portfolio ZIP & Extract
  1. Download the ZIP file from PortSuite (button above)
  2. Right-click the ZIP → "Extract All"
  3. Choose a folder (e.g., Desktop). Remember the exact folder name!
6
Upload Using Terminal (Copy-Paste 6 Commands)
Open Terminal/PowerShell and paste these commands one by one, pressing Enter after each. Click the copy icon (📋) next to each command:
Command 1 - Go to your extracted folder:
cd Desktop/YOUR-FOLDER-NAME
Replace YOUR-FOLDER-NAME with the name of your extracted portfolio folder
Command 2 - Clone your repository:
git clone https://github.com/YOUR-USERNAME/YOUR-USERNAME.github.io.git
Replace YOUR-USERNAME with your GitHub username
Command 3 - Go into the repo folder:
cd YOUR-USERNAME.github.io
Replace YOUR-USERNAME with your GitHub username
Command 4 - Configure Git (First time only):
Run these 2 commands to tell Git who you are:
A)
git config --global user.name "Your Name"
B)
git config --global user.email "your@email.com"
Replace "Your Name" and "your@email.com" with your actual name and email
Command 5 - Copy portfolio files:
cp -r ../YOUR-FOLDER-NAME/* .
Replace YOUR-FOLDER-NAME with your extracted portfolio folder name
Command 6a - Fix branch name:
git branch -M main
(Fixes: "src refspec main does not match any" error)
Command 6b - Add files:
git add .
Command 6c - Upload to GitHub:
git commit -m "Upload portfolio" && git push -u origin main
When asked for password, paste your token from step 3
✅ Success! If no red errors appear, your portfolio is uploading! Wait 2-3 minutes, then visit https://YOUR-USERNAME.github.io
Important File Size Warning:
GitHub does NOT allow files larger than 100MB. If your portfolio contains large videos or images, upload them to Google Drive or YouTube and link them instead. Your current portfolio is fine, but keep this in mind if you add media later.
7
Your Portfolio is LIVE!
Visit https://YOUR-USERNAME.github.io to see your live portfolio! Share this link with recruiters and clients.
❓
Troubleshooting
Error: "command not found"
→ Git not installed. Download from git-scm.com and restart terminal.

Error: "Please tell me who you are"
→ Run Command 4a and 4b first

Error: "src refspec main does not match any"
→ You skipped Command 6a. Run: git branch -M main

Error: "permission denied"
→ Try with sudo (for Mac/Linux only)

Files not showing on website?
→ Check the homepage file is named index.html
1
Create a free GitHub account
Go to github.com and sign up. Username will be your portfolio URL.
2
Create repository
Click + → New Repository. Name: username.github.io. Set to Public. Do NOT check "Add a README file". Create Repository.
3
Upload files
In repository, click "Add file → Upload files". Drag extracted portfolio files. Commit with message "Initial upload".
4
Enable GitHub Pages
Settings → Pages. Source: Deploy from a branch. Branch: main, Folder: / (root). Save.
5
Done! 🎉
